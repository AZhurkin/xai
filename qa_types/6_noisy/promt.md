# Задача:
На основе предоставленного фрагмента текста на русском языке сформировать пары *вопрос–ответ*
с обязательным указанием *evidence_span* для доказательства корректности ответа.
Должен быть **дисфлюентный (шумный) вопрос на русском языке**,
**не менее 3 пар *вопрос–ответ***, каждая из которых основана на **разных фактах**!

# Инструкции:

1. Внимательно прочитай исходный текст на русском языке и используй **исключительно**
   информацию, содержащуюся в нём.
2. Выбери **как минимум три различных явно сформулированных факта, правила или утверждения**,
   присутствующих в тексте и подходящих для проверки корректности извлечения информации.
   * Каждый факт должен быть независимым от остальных.
3. Для **каждого факта отдельно** сформулируй вопрос и ответ.
4. **Вопрос в стиле человека 18–25 лет**, который пишет быстро с телефона и допускает много ошибок, соблюдая два ключевых условия:
   * **Смысл должен оставаться тем же**, чтобы ответ не менялся.
   * Вопрос должен быть **однозначно привязан** к соответствующему месту в `context` (без расплывчатых «это/то»).
5. Требования к «шумности» формулировки (используй умеренно, но явно):
   * опечатки, пропуски или лишние буквы (например: «обязнности», «отвчает», «правилоо»);
   * неправильные окончания/падежи, разговорная грамматика;
   * разговорные вставки: «типа», «короч», «вобщем», «прям», «блин»;
   * сокращения: «щас», «непон», «чё», «какта», «пж», «плиз» (только русская запись, без латиницы);
   * отсутствие знаков препинания или хаотичное использование;
   * повтор букв для эмоций: «нууу», «зачеееем», «почемууу»;
   * эмодзи **иногда** (не более 1–2 за вопрос), чтобы не ломать читаемость.
6. Ограничения на качество и однозначность (обязательные):
   * несмотря на ошибки, вопрос должен оставаться **понятным человеку** с учётом `context`;
   * не добавляй новых условий, фактов или уточнений, которых нет в `context`;
   * не делай вопрос более общим или более узким по смыслу;
   * не используй иностранные слова, транслитерацию и латиницу;
   * не используй оскорбления, ненормативную лексику, уничижительные выражения;
   * избегай местоименных ссылок без явной привязки (например, «кто это?» без указания сущности).
7. Сформулируй **краткий и точный ответ**, строго на основе текста:
   * Ответ отражает только информацию, явно присутствующую в документе.
   * Запрещено добавлять домыслы, интерпретации или внешние знания.
   * Допускается краткое переформулирование без искажения смысла.
   * Ответ не должен быть избыточным или содержать длинные цитаты.
   * Ответ полностью на русском языке.
   * Ответ однозначно подтверждается evidence span.
8. **Evidence span**:
   * `evidence_span` — это **дословная цитата из исходного текста**, которая **непосредственно доказывает корректность ответа**.
   * Evidence span должен быть **непрерывным фрагментом текста**.
   * Цитата должна быть **минимально достаточной** для подтверждения ответа.
   * Запрещено:
     * изменять текст цитаты;
     * сокращать её многоточиями;
     * объединять несколько несмежных фрагментов;
     * добавлять пояснения вне цитаты.
9. Поле `"type"` **всегда** заполняй значением `"noisy"`.
10. **Формат вывода строго `jsonl`**, без комментариев и дополнительного текста.
   Каждая строка — один объект.

# Формат:
```json
{"question": "...", "answer": "...", "evidence_span": "...", "type": "noisy"}
```

# Ограничения:
    * Один вопрос — один факт.
    * Никакой информации вне текста.
    * Строгое соблюдение формата.
